できること

GitHub Pagesは紹介ページです。実際の監視・通知・DB保存は、手元のPCやサーバーでPythonアプリを起動して使います。

発売情報を取得

公式サイトの発売情報を取り込み、商品名・発売日・URLをSQLiteへ保存します。

店舗ページを監視

ショップの商品ページや検索ページから、在庫や抽選受付の文言を検知します。

Discord通知

購入チャンスを検知したらWebhook経由で通知し、商品ページをすぐ開けます。

手動確認で安全運用

購入確定はユーザーが確認します。CAPTCHA回避や規約回避は対象外です。

監視フロー

1発売情報

公式・補助サイトから新商品の情報を取り込みます。

2DB保存

商品、店舗、監視結果、通知履歴をローカルDBへ保存します。

3在庫確認

設定した店舗URLを一定間隔でチェックします。

4通知

購入・抽選の可能性があればDiscordへ通知します。

5購入確認

最後はブラウザで内容を確認して購入します。

ローカル起動

リポジトリをダウンロードしたあと、`config.example.json` を `config.json` にコピーしてDiscord Webhookや監視店舗を設定します。 

Copy-Item config.example.json config.json
python app.py init
python app.py serve

# 管理画面
http://127.0.0.1:8765

# 継続監視
python app.py watch
運用ポリシー

このツールは、発売情報の整理、在庫・抽選ページの監視、通知、購入ページへの誘導を目的にしています。店舗の購入制限回避、CAPTCHA回避、複数アカウント運用、規約に反する自動購入は対象外です。
